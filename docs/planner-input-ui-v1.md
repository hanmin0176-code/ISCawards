# 설계사 입력 화면 구조 v1

## 목표
설계사가 복잡한 JSON이나 시상표 원문을 보지 않고,
자신의 실적만 입력하면
- 익월 예상 시상
- 13회차 이후 예상 시상
- 연속가동 충족 여부
- 여행 시상 충족 여부
를 확인할 수 있게 한다.

---

## 화면 구성

### 1. 로그인 후 첫 화면: 설계사 대시보드
표시 항목
- 이번달 입력한 실적 건수
- 다음달 예상 지급 총액
- 13회차 대기 총액
- 달성 중인 연속가동 규칙 수
- 여행 시상 가능성 알림

바로가기 버튼
- `실적 입력`
- `예상 시상 보기`
- `시상표 조회 채팅`

---

### 2. 실적 입력 화면

## 입력 방식 A. 빠른 입력
초기 MVP는 이 방식을 우선 추천

입력 항목
- 보험사
- 연도
- 월
- 주차
- 실적 기간 자동표시
- 상품군별 입력행

상품군별 입력 예시
- 인보험 월납보험료 합계
- 펫보험 월납보험료 합계
- 캣보험 월납보험료 합계
- The라이프업 월납보험료 합계
- 기타 필요 상품군

화면 예시
- 인보험: 300000
- 펫보험: 120000
- 캣보험: 0
- The라이프업: 0

버튼
- `저장`
- `예상 시상 계산`

장점
- 빠르게 입력 가능
- 익월/13회차 기본 계산 가능

한계
- 계약별 유지 추적 정밀도는 낮음

---

## 입력 방식 B. 계약 기준 입력
정밀 계산용

입력 항목
- 계약일
- 상품명
- 상품군
- 월납보험료
- 계약번호(비식별)
- 유지 필요 여부

장점
- 13회차 유지 계산 정확도 높음
- 환수 추적 가능

권장
- MVP 이후 2차 단계에서 도입

---

### 3. 예상 시상 결과 화면
입력 완료 후 즉시 보여줄 항목

#### A. 익월 예상 시상
- 규칙명
- 기준 실적
- 달성 구간
- 예상 금액
- 지급 예상 월

예시
- 인보험 익월 400%: 상품별 월납보험료 기준 계산 필요
- 2주차 인보험 익월 현금 시상: 90만원

#### B. 13회차 이후 예상 시상
- 규칙명
- 유지 조건
- 예상 금액
- 상태 (`대기`, `유지확인필요`, `확정예정`)

예시
- 2주차 현금추가: 60만원
- Prestige CLUB: 180만원

#### C. 연속가동 / 여행 시상
- 현재 충족 중인 조건
- 아직 부족한 조건
- 달성 시 받을 보상

예시
- 코타키나발루 조건1: 4월, 5월 조건 중 1개 남음
- 푸켓+카오락: 4월 실적 30만원 부족

---

### 4. 얼마 더 하면 달성 화면
목표
- 사용자가 추가로 얼마를 더 넣으면 다음 구간 달성인지 보여주기

표시 항목
- 현재 실적
- 다음 목표 구간
- 부족 금액
- 달성 시 추가 시상
- 관련 규칙명

예시
- 인보험 20만 구간까지 4만원 부족
- 달성 시 익월 현금 30만원 추가 가능

---

### 5. 시상표 조회 채팅
질문 예시
- `2026년 4월 2주차 삼성화재 시상 알려줘`
- `2주차 인보험 익월지급 얼마야?`
- `삼성화재 여행 시상 조건 뭐야?`

답변 원칙
- DB에 저장된 확정 규칙만 답변
- 불명확한 부분은 `[추가확인필요]`
- 계산이 필요한 질문은 입력 데이터와 연결해서 답변

---

## 추천 MVP 화면 순서

1. 대시보드
2. 실적 입력
3. 예상 시상 결과
4. 시상표 조회 채팅

`얼마 더 하면 달성`은 2차로 추가

---

## 사용자 경험 원칙

1. JSON은 설계사에게 보이지 않는다.
2. 용어는 `익월 예상`, `13회차 대기`, `여행 시상`처럼 직관적으로 쓴다.
3. 규칙명과 근거를 같이 보여준다.
4. 불명확한 여행 조건은 무리해서 확정하지 않는다.

---

## 추가 확인 필요

- 실적 입력을 주차 단위만 받을지, 계약 단위까지 함께 받을지
- 펫/캣/The라이프업 등 상품군 분류 기준을 어떤 테이블에서 관리할지
- 설계사 개인 계정 생성 방식을 Supabase auth로 바로 할지 별도 직원 테이블과 연결할지
