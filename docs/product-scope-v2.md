# ISCawards 제품 범위 재정의 v2

## 1. 이번 대화 기준으로 확정할 목표

현재 프로젝트의 1차 목표는 아래 2가지다.

### 목표 A. 설계사용 시상 안내
설계사가 로그인해서
- 몇 년 몇 월 몇 주차에
- 어느 보험사에
- 얼마의 실적을 넣었는지
입력하면,

시스템이
- 익월 지급 예정 시상
- 13회차 납입 후 지급 예정 시상
- 연속가동 시상
- 여행 시상 달성 여부
를 알려준다.

### 목표 B. 과거 시상 조회형 채팅
사용자가 자연어로
- `2026년 4월 2주차 삼성화재 시상 뭐였어?`
- `삼성화재 2주차 인보험은 익월 몇 퍼센트였어?`
- `4월 2주차 여행 시상 조건 알려줘`
처럼 물으면,

DB에 저장된 확정 시상 규칙을 검색해서 답변한다.

---

## 2. 현재 AI 출력 방식의 문제

현재 AI는 시상표 이미지를 읽고 아래처럼 너무 느슨한 구조를 반환하고 있다.

- `tier`
- `details`

이 구조는 사람이 보기에도 애매하고, 계산 로직이 직접 쓰기 어렵다.

예를 들어 아래 문제가 있다.

1. `2주차 인보험 / 익월(5월) 지급` 같은 섹션명이 빠짐
2. `10만 -> 30만 / 20만 -> 60만 ...` 같은 구간 배열이 `tiers`로 정리되지 않음
3. `익월 지급`, `13회차 유지`, `13회차 미유지시 환수`가 분리되지 않음
4. 여행 시상처럼 `조건1 또는 조건2`, `택1` 구조를 표현하지 못함
5. `월납보험료의 n%` 지급 규칙과 `현금 고정금액` 규칙이 섞여 표현됨

즉, 앞으로는 AI 출력 목표를 `사람이 대충 읽는 JSON`이 아니라 `계산 가능한 규칙 JSON`으로 바꿔야 한다.

## 3. 앞으로 사용할 규칙 타입

시상표는 아래 5가지 타입으로 분리한다.

### 3-1. percentage_payout
월납보험료의 몇 퍼센트를 익월 지급하는 규칙

예:
- 인보험 익월 400%
- 펫/캣/The라이프업 익월 300%

### 3-2. tiered_cash_next_month
특정 기간 실적 구간별로 익월 현금을 주는 규칙

예:
- 10만 달성 -> 30만
- 20만 달성 -> 60만
- 30만 달성 -> 90만
- 50만 달성 -> 150만

### 3-3. tiered_cash_after_13th
특정 기간 실적 구간별로 13회차 납입 후 현금을 주는 규칙

예:
- 10만 달성 -> 20만
- 20만 달성 -> 40만
- 30만 달성 -> 60만
- 50만 달성 -> 100만

### 3-4. consecutive_bonus
전월/당월/차월 연속 조건을 달성하면 주는 규칙

예:
- 26.3~4월 연속가동 시상
- Prestige CLUB 4월~6월 연속

### 3-5. trip_or_choice_reward
조건1 또는 조건2를 충족하면 여행/현금 택1 시상을 주는 규칙

예:
- 코타키나발루
- 럭셔리 푸켓+카오락
- 헝가리+오스트리아 또는 럭셔리 싱가포르
- 캐나다

## 4. 이 시상표를 사람이 해석한 기준 정리

아래 내용은 이번 대화에서 사용자가 직접 설명한 해석을 우선 반영한다.
단, 이미지 원문과 차이가 있을 수 있는 부분은 `[추가확인필요]`로 본다.

### 규칙 1. 인보험 시상 익월 지급
- 대상: 인보험
- 제외: 실손보험, 단체보험, 펫보험, 캣보험, The라이프업보험
- 기준: 월납보험료
- 지급: 익월
- 지급률 합계: 400%
  - 삼성화재 기본 100%
  - 삼성화재 추가 100%
  - 글로벌 자체시상 200%

### 규칙 2. 펫/캣/The라이프업 익월 지급
- 대상: 펫보험, 캣보험, The라이프업보험
- 기준: 월납보험료
- 지급: 익월
- 지급률: 300%

### 규칙 3. 2주차 인보험 / 익월(5월) 지급
- 실적기간: 4.6 ~ 4.12
- 지급시점: 익월(5월)
- 구간별 시상
  - 10만 -> 30만
  - 20만 -> 60만
  - 30만 -> 90만
  - 50만 -> 150만
- 13회차 미유지 시 환수 규정 존재

### 규칙 4. 2주차 현금추가 / 13회차 유지
- 실적기간: 4.6 ~ 4.12
- 지급시점: 13회차 납입 후
- 구간별 시상
  - 10만 -> 20만
  - 20만 -> 40만
  - 30만 -> 60만
  - 50만 -> 100만

### 규칙 5. 26.3~4월 연속가동 시상 / 13회차 유지
- 전월 기준 구간 달성 후
- 4.1 ~ 4.19 기간에 10만 실적 충족 시
- 전월 구간별 보상 지급
  - 10만 구간 -> 20만
  - 20만 구간 -> 60만
  - 30만 구간 -> 80만
  - 50만 구간 -> 180만
- 13회차 납입 후 지급
- 일부 세부 조건은 원문 재검수 필요

### 규칙 6. 여행 시상 / 조건1 또는 조건2
예시 구조
- 코타키나발루
  - 조건1: 26.3.16~31 = 10만, 26.4월 = 20만, 26.5월 = 20만
  - 조건2: 26.4월 = [추가확인필요], 26.5월 = 20만
  - 시상: 코타키나발루 1명
- 럭셔리 푸켓+카오락
  - 조건1: 26.3.16~31 = 20만, 26.4월 = 30만, 26.5월 = 30만
  - 조건2: 26.4월 = 60만, 26.5월 = 30만
  - 시상 택1
    - 푸켓+카오락 1명
    - 코타키나발루 1명 + 70만

기타 여행 시상도 같은 구조로 본다.

### 규칙 7. 삼성 Prestige CLUB
- 기간: 4월, 5월, 6월 연속
- 지급조건: 13회차 납입
- 구간별 보상
  - 20만 -> 60만
  - 40만 -> 120만
  - 60만 -> 180만
  - 80만 -> 240만
  - 100만 -> 300만
  - 120만 -> 360만
  - 150만 -> 450만

## 5. 앞으로 AI가 목표로 해야 할 출력 구조

```json
{
  "schema_version": "2.0.0",
  "campaign_meta": {
    "insurer_code": "samsung_fire",
    "insurer_name": "삼성화재",
    "campaign_name": "26.4월 2주차 삼성화재 시상",
    "campaign_year": 2026,
    "campaign_month": 4,
    "week_label": "2주차"
  },
  "rules": [
    {
      "rule_code": "week2_life_percentage",
      "rule_type": "percentage_payout",
      "rule_name": "인보험 시상 익월지급",
      "target_products": {
        "include": ["인보험"],
        "exclude": ["실손보험", "단체보험", "펫보험", "캣보험", "The라이프업보험"]
      },
      "base_metric": "monthly_premium",
      "payout_timing": "next_month",
      "percentage_total": 400,
      "percentage_breakdown": [
        { "label": "삼성화재 기본", "percent": 100 },
        { "label": "삼성화재 추가", "percent": 100 },
        { "label": "글로벌 자체시상", "percent": 200 }
      ]
    },
    {
      "rule_code": "week2_life_cash_next_month",
      "rule_type": "tiered_cash_next_month",
      "rule_name": "2주차 인보험 익월지급",
      "sales_periods": [
        { "start_date": "2026-04-06", "end_date": "2026-04-12" }
      ],
      "tiers": [
        { "threshold": 100000, "reward_cash": 300000 },
        { "threshold": 200000, "reward_cash": 600000 },
        { "threshold": 300000, "reward_cash": 900000 },
        { "threshold": 500000, "reward_cash": 1500000 }
      ],
      "clawback_rule": "13회차 미유지 시 환수"
    }
  ]
}
```

핵심은 다음이다.

- `sections` 중심이 아니라 `rules` 중심
- 계산 가능한 필드는 반드시 분리
- `percentage_payout`와 `fixed_cash_tiers`를 따로 구분
- 여행 시상은 `condition_sets`와 `reward_options`로 분리

## 6. 기능 구현 우선순위

### 1차 MVP
1. 관리자 시상표 업로드
2. AI 초안 생성
3. 관리자 검수/확정
4. 설계사 로그인
5. 실적 입력
6. 익월 시상 / 13회차 시상 계산
7. 과거 시상 조회형 채팅

### 2차
1. 여행 시상 조건 계산
2. 연속가동 보상 계산
3. Prestige CLUB 계산
4. 얼마 더 하면 달성되는지 안내

## 7. 설계사 화면에서 보여줄 답변 형태

### 예시 A. 실적 입력 결과
- 2026년 4월 2주차 삼성화재 인보험 실적 30만원 입력
- 익월 지급 예상
  - 월납보험료 기준 400% 시상: 상품별 계산 필요
  - 2주차 인보험 익월 현금 시상: 90만원
- 13회차 납입 후 추가 지급 예상
  - 2주차 현금추가: 60만원

### 예시 B. 채팅 조회
질문:
- `2026년 4월 2주차 삼성화재 인보험 익월 시상 뭐야?`

답변:
- 인보험은 월납보험료의 400% 익월 지급
- 별도로 4.6~12일 실적 구간 시상은
  - 10만 -> 30만
  - 20만 -> 60만
  - 30만 -> 90만
  - 50만 -> 150만
- 13회차 미유지 시 환수 규정 확인 필요
